üìã What Each Prompt Does

Project Builder V1

Phase What Happens Human Action
Phase 0: Input Scans for existing code, analyzes materials (PDFs, docs, images), asks questions Provide materials or describe project
Phase 1: Planning Creates REQUIREMENTS.md, ARCHITECTURE.md, TASKS.md APPROVE before it builds
Phase 2: Implementation Builds each task: tests first, then code, then commit None ‚Äî fully autonomous
Phase 3: Finalization Creates docs, E2E tests, deployment instructions Deploy + run E2E tests yourself

QA Audit V4

Phase What Happens Human Action
Phase 1: Reconnaissance Reads package.json, scans structure, detects project type (HIPAA, CROA, etc.) None
Phase 2: Audit All 6 specialists audit: Security, Legal, DevOps, QA, Accessibility, UI/UX None
Phase 3: Fix Implements ALL fixes (Critical ‚Üí High ‚Üí Medium ‚Üí Low), commits each None
Phase 4: Test Runs TypeScript, lint, unit tests; loops until green Run E2E tests yourself
Phase 5: Report Creates QA_AUDIT_REPORT.md, pushes to GitHub Review report
